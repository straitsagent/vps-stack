# Earnings Report Standards

These apply to **every** earnings analysis report (pre and post) generated by `portfolio_earnings_analysis.py`. When modifying the script, ensure all six standards remain in place.

1. **Date header** — every report must show both date of writing and expected/actual earnings date:
   `**Date written:** YYYY-MM-DD | **Expected/Actual earnings:** YYYY-MM-DD`

2. **Portfolio position summary** — always include shares held, latest price, total position value in USD, and % weight of total portfolio. Sourced from `price_history` + `fx_rates` tables in real time.

3. **Company/research overview** — always include a synopsis of the most recent `research_reports` entry (depth + date noted). If no research report exists, seed a 150-word overview via Exa + Grok before synthesis.

4. **Sources list** — every report footer lists all primary sources consulted (EDGAR 8-K, Exa transcript, Finnhub, yfinance, etc.) with a count.

5. **Token usage and cost** — every report footer shows: Model name, input tokens, output tokens, estimated USD cost (Grok-4.3: $1.25/1M input + $2.50/1M output).

6. **Email recipient** — all earnings reports go to `<YOUR_RECIPIENT_EMAIL>` (consistent with all other workflow emails). Never send to `smtp_cfg["username"]`.
